Classes

A principal diferença entre as linguagens C e C++ é a orientação a objetos presente em C++. Neste breve guia, veremos o que são classes e como instanciá-las, na linguagem C++, para a criação de objetos.
Através da programação orientada a objetos (POO), é possível modelar com maior precisão o mundo real computacionalmente.

Considere o seguinte cenário: Necessita-se registrar um cliente em um sistema bancário. Para tal registro, será necessário o nome, o CPF e RG do cliente. O cliente possuirá uma conta bancária, ou seja, informações básicas como o saldo, por exemplo, também devem ser registradas. A conta bancária possuirá também a simples função de realizar transferências.

Ainda neste exemplo bastante simplório, percebe-se que podem existir muitos clientes, portanto, é impraticável imaginar uma implementação desse cenário em que manualmente fosse criada uma variável para cada um desses atributos, certo? Neste caso, a programação orientada a objetos mostra-se bastante eficiente e o problema pode ser facilmente contornado utilizando uma abstração de todas as informações que uma conta bancária possui.

Em orientação a objetos, uma classe é uma descrição que abstrai um conjunto de objetos com características similares[1].

Podemos criar uma classe Conta. Essa classe possuirá, como atributos, nome, CPF, RG e saldo.
Há um problema com a definição acima: Os atributos estão todos com nível de acesso definido como public e podem facilmente ser acessados e modificados em qualquer parte do programa. É um problema, pois são informações importantes que devem ser minimamente protegidas.

Alteramos, portanto, o nível de acesso de public para private. O ato de definir o tipo de acesso ao conteúdo das classes compõe um dos pilares da POO: Encapsulamento.
Devemos manter o acesso a um atributo sempre como private, a menos que tenhamos um bom motivo para deixá-lo com outro nível de acesso[2].

Getters e Setters
Com os atributos definidos como private, como acessá-los quando necessário?
Neste momento, entram os métodos Setters e Getters, que são métodos responsáveis, respectivamente, por modificar os atributos de uma classe e mostrá-los (retorná-los). Além disso, através dos setters, podemos definir restrições para a modificação dos atributos de um objeto. Não há restrição no exemplo citado, porém há casos em que isso torna-se bastante necessário.

Criando um método
É possível definir funções que o objeto de uma classe poderá executar, as quais são chamadas de métodos. Vamos implementar a função de transferência entre contas bancárias. Para tal, criaremos o método transferir().
