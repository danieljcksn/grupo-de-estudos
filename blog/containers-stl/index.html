É bastante comum, seja no desenvolvimento ou na resolução de problemas de programação, necessitarmos manipular dados; Pesquisa, busca, inserção e remoção de informações em uma coleção de dados são exemplos de operações recorrentes que todo desenvolvedor utiliza na criação de algoritmos, e para tal existem diversas estruturas de dados: Fila, Lista e Pilha, por exemplo, possuem vasta utilização.

Implementar cada uma dessas estruturas de dados possui custo de tempo elevado, afinal, todas envolvem a utilização de ponteiros, e a construção desses algoritmos exige do programador muita atenção e tempo para que problemas não ocorram. Ter essas estruturas (e outras) implementadas previamente, de modo que sejam facilmente instanciadas ajuda, certamente, na utilização eficaz do tempo e energia do programador. Felizmente, na linguagem C++, isso é uma realidade: STL!

A Standard Template Library (STL) é uma biblioteca padronizada de funções. Esta biblioteca é composta por quatro componentes[1]:

  •    • Contêineres
  •    • Algoritmos
  •    • Iteradores
  •    • Functores


Veremos, neste post, um pouco sobre os contêineres (containers) encontrados em C++.


Contêineres - O que são?

Os contêineres são algoritmos pré-implementados que representam estruturas de dados auxiliares através de classes template. Por serem classes template, tais contêineres podem ser utilizados para armazenar elementos de quaisquer tipos de dados, bem como outras classes[2].
Cada contêiner administra o armazenamento necessário para seus elementos e oferece acesso para cada elemento através de iteradores e/ou funções-membro[3].

Existem 4 tipos de contêiner em C++, são eles:

  •    1. Sequenciais
  •    2. Adaptadores
  •    3. Associativos
  •    4. Associativos Desordenados

Vejamos um pouco sobre cada um desses contêineres.


1. Contêineres Sequenciais

Os contêineres sequenciais são representações de estruturas de dados que armazenam elementos homogêneos, de modo que cada elemento do contêiner pode ser acessado sequencialmente.
Os contêineres sequenciais presentes na atual versao do C++ são array, vector, deque, forward_list e list.


1.1 Array

O contêiner array representa um vetor estático e contínuo (os elementos são alocados sequencialmente na memória, mas o número máximo de elementos não pode ser alterado). Esse contêiner foi criado com a finalidade de ser uma releitura do array presente na linguagem C, porém com a mesma performance e maior legibilidade.
A interface do array presente na STL torna mais conveniente as operações para encontrar o tamanho do array e copiar elementos[4], por exemplo, bem como permite a utilização de algoritmos baseados em iteradores, definidos na STL (falaremos sobre iteradores no próximo post).

1.2 Vector

O contêiner sequencial Vector é baseado em arrays e suporta acesso aleatório a seus elementos[5]. Ele é capaz de reproduzir todas as características do array e, como principal diferença, possui a capacidade de expandir a quantidade de elementos dinamicamente a cada inserção realizada. Esse contêiner é contínuo (os elementos de um Vector são alocados sequencialmente na memória, e são individualmente acessíveis, por esta razão, através do operador [ ] ) e dinâmico (é capaz de redimensionar seu tamanho automaticamente, de acordo com a quantidade de elementos inseridos)[6].
Vectors também podem alocar armazenamento extra na memória e por esta razão alguns possuem uma capacidade de armazenamento superior ao necessário para o armazenamento dos itens. A capacidade pode ser encontrada através da função-membro capacity(). Portanto, comparado à arrays, vectors consomem mais memória em troca da habilidade de controlar o armazenamento e crescer dinamicamente de forma eficiente[7].

O Vector possui suporte a funções-membro bastante semelhantes às que foram vista anteriormente, e também possui funções próprias. push_back( ): Insere um elemento no final do vector.
pop_back( ): Retira último elemento do vector.
erase( ): Remove um elemento (posição) ou um intervalo de elementos ([primeiro, último))
clear( ): Elimina todos os elementos do Vector

Bem como o array, Vectors também possuem as funções membro at, front, back, swap, empty, size, begin, end.

1.3 Deque

Deque é uma implementação de filas duplamente terminadas (Double Ended Queue).
Nesse contêiner, os elementos podem ser inseridos tanto na frente (cabeça) quanto no fim (cauda)[7]. Deques possuem funcionalidades bastante semelhantes à Vectors, porém com eficiente inserção e remoção de elementos também no início da sequência, não apenas no fim (como o Vector). No entanto, diferente de Vectors não há a garantia de que todos os elementos serão armazenados em posições contínuas da memória. Para operações que envolvem frequentes inserções e remoções de elementos localizados em posições diferentes do início e fim, deques possuem desempenho pior e iteradores menos consistentes, comparado à listas encadeadas / duplamente encadeadas.[8]

Principais funções-membro:
→ push_front( ): Insere um elemento no início.
→ push_back( ): Insere um elemento no fim.
→ pop_front( ): Remove o primeiro elemento
→ pop_back( ): Remove o último elemento.

A deque também possui algumas funções-membro e operadores que vectors e arrays possuem. Para mais detalhes, acesse a página C Plus Plus.

1.4 Foward List

Forward lists são containers sequenciais que permitem operações de inserção e remoção de elementos em qualquer posição da sequência em tempo constante.
Forward lists são implementadas semelhantemente à listas simplesmente encadeadas; Listas simplesmente encadeadas podem armazenar cada um de seus elementos em alocações diferentes de memória. Para manter a ordem, cada elemento possui um ponteiro para o próximo elemento da sequência.
Comparado aos outros contêineres sequenciais (array, vector e deque), forward_list tem melhor performance na inserção, extração e na movimentação de elementos em qualquer posição dentro do contêiner, assim como em algoritmos que usam essas operações intensivamente, como algoritmos de ordenação, por exemplo.
A principal desvantagem de foward_lists e lists, quando comparadas aos outros contêineres sequenciais é que estes oferecem acesso direto aos elementos a partir da posição deles. Por exemplo, para acessar o sexto elemento em uma foward_list, será necessário iterar do início do contêiner até aquela posição, feito em tempo linear determinado pela distância entre esses elementos. Foward_lists também consomem memória extra para manter as informações de linkagem da lista[9].


Principais funções-membro:
assign( ): Determina os novos elementos que irão compor a lista, substituindo o conteúdo atual e reajustando o tamanho de acordo a quantidade determinada.
push_front( ): Insere um elemento no início.
pop_front( ): Remove primeiro elemento.
insert_after( ):Insere um ou mais elementos depois de uma posição dada.
erase_after( ):Elimina um elemento ou um intervalo de elementos a partir de uma posição dada.

1.5 List

Lists são contêineres sequenciais que permitem operações de inserção e remoção de elementos em quaisquer posições na sequência, bem como permitem a iteração em ambas direções.
Esses contêineres são implementados do mesmo modo que listas duplamente encadeadas; Listas duplamente encadeadas podem armazenar cada um de seus elementos em posições de memória diferentes e não-relacionadas. Para a ordem ser mantida, são armazenados também ponteiros que associam cada elemento ao seu antecessor e ao seu sucessor.


Principais funções-membro:
assign( ): Determina os novos elementos que irão compor a lista, substituindo o conteúdo atual e reajustando o tamanho de acordo a quantidade determinada.
push_front( ): Insere um elemento no início.
push_back( ): Insere um elemento no fim.
pop_front( ): Remove o primeiro elemento.
pop_back( ): Remove o último elemento
insert( ): Insere um ou mais elementos depois de uma posição dada.
erase( ): Elimina um elemento ou um intervalo de elementos a partir de uma posição dada.

2. Adaptadores

Adaptadores de contêiner são construídos através da adaptação de contêineres sequenciais, restrigindo e/ou modificando as interfaces destes para oferecer o comportamento desejado[10].


2.1 Stack

Stacks são um tipo de adaptador de contêiner, projetados para executarem o comportamento LIFO (Last-in, first-out), onde os elementos são inseridos e retirados do topo da estrutura, exatamente a ideia oferecida pela Pilha. Stack é uma adaptação do contêiner deque.

Principais funções-membro:
empty( ): Retorna true se o tamanho da stack for zero e false, caso contrário.
size( ): Retorna o tamanho da stack.
top( ): Retorna o elemento localizado no "topo" da stack.
push( ): Insere um novo elemento no topo da stack, sobre o atual top.
pop( ): Retira o elemento localizado no topo da stack.

Este adaptador foi projetado para executar o padrão FIFO (fist-in, first-out), uma implementação da estrutura de dados Fila. Este contêiner é uma adaptação do deque.

Principais funções-membro:
empty( ): Retorna true se o tamanho da queue for zero e false, caso contrário.
size( ): Retorna o tamanho da queue.
front( ): Retorna o elemento primeiro elemento da queue.
back( ): Acessa o último elemento.
push( ): Insere um elemento.
pop( ): Retira o primeiro elemento.
